---
title: "Подключение LLM-модели для оценки в Doc Reviewer"
description: "Настройте OpenAI, Anthropic, Ollama или любую совместимую с OpenAI модель. Все API-ключи хранятся локально и не передаются в Doc Reviewer."
---

Doc Reviewer делегирует всю работу по оценке большой языковой модели. До первой оценки вам нужно подключить хотя бы одну модель. Управление моделями доступно в **Настройках → Модели**: вы добавляете, редактируете, активируете и удаляете конфигурации моделей. Для оценки используется только активная модель, но вы можете хранить несколько конфигураций и переключаться между ними в любое время.

## Поддерживаемые провайдеры

| Провайдер | Base URL | API-ключ обязателен |
|---|---|---|
| OpenAI | `https://api.openai.com/v1` | Да |
| Anthropic | `https://api.anthropic.com/v1` | Да |
| Ollama (локально) | `http://localhost:11434/v1` | Нет |
| Любой совместимый с OpenAI API | Ваш пользовательский Base URL | Зависит от провайдера |

<Warning>
Качество модели напрямую влияет на точность оценки. Используйте мощную модель — GPT-4o, Claude 3.5 Sonnet или аналогичную — для надёжных результатов. Более слабые или маленькие модели могут пропускать проблемы или давать расплывчатые рекомендации.
</Warning>

## Добавление модели

<Steps>
  <Step title="Откройте Настройки">
    Нажмите **Настройки** в верхней навигации, затем выберите вкладку **Модели**.
  </Step>
  <Step title="Нажмите «Добавить модель»">
    Нажмите кнопку **Добавить модель**, чтобы открыть форму настройки модели.
  </Step>
  <Step title="Заполните параметры модели">
    Заполните все обязательные поля:

    - **ID модели** — идентификатор, который API ожидает, например `gpt-4o` или `claude-3-5-sonnet-20241022`. Он должен точно совпадать с названием модели, используемым в API вашего провайдера.
    - **Отображаемое имя** — читаемое название, отображаемое в интерфейсе, например `GPT-4o`.
    - **Провайдер** — выберите провайдера или введите пользовательское значение.
    - **Base URL** — конечная точка API. Используйте значения из таблицы выше или документацию вашего провайдера.
    - **API-ключ** — вставьте секретный ключ. Для провайдеров, не требующих ключа (например, Ollama), оставьте поле пустым.
  </Step>
  <Step title="Сохраните модель">
    Нажмите **Сохранить**. Модель появится в списке, но не будет активной.
  </Step>
  <Step title="Активируйте модель">
    Нажмите **Активировать** рядом с нужной моделью. Активная модель используется для всех последующих оценок.
  </Step>
</Steps>

<Note>
В один момент времени может быть активна только одна модель. Переключение активной модели вступает в силу немедленно — следующая оценка будет использовать вновь активированную модель.
</Note>

## Переключение активной модели

Чтобы переключиться на другую модель, перейдите в **Настройки → Модели** и нажмите **Активировать** рядом с нужной моделью. Ранее активная модель деактивируется автоматически.

## Безопасность API-ключей

<Note>
API-ключи хранятся в локальной базе данных SQLite на вашем компьютере. Они не передаются на серверы Doc Reviewer — таких серверов не существует. Единственный исходящий трафик с API-ключом — прямой API-вызов с вашего компьютера к провайдеру LLM.
</Note>

## Использование локальной модели через Ollama

Ollama позволяет запускать модели с открытым весом локально, без API-ключа и внешнего сервиса. Это удобно, если вы хотите держать документы на своей инфраструктуре или избежать поминутной оплаты.

<Steps>
  <Step title="Установите Ollama">
    Скачайте и установите Ollama с [ollama.com](https://ollama.com). После установки Ollama работает как фоновый сервис и слушает порт `11434`.
  </Step>
  <Step title="Загрузите модель">
    Откройте терминал и запустите модель, чтобы скачать её:

    ```bash
    ollama run llama3
    ```

    Вы можете использовать любую модель из [библиотеки Ollama](https://ollama.com/library). Команда `ollama run` скачает модель, если она ещё не загружена, и запустит интерактивный сеанс — его можно закрыть после завершения загрузки.
  </Step>
  <Step title="Добавьте модель в Doc Reviewer">
    Перейдите в **Настройки → Модели → Добавить модель** и заполните:

    - **ID модели** — название модели в Ollama, например `llama3` или `mistral`.
    - **Отображаемое имя** — любое удобное вам название.
    - **Base URL** — `http://localhost:11434/v1`
    - **API-ключ** — оставьте пустым.
  </Step>
  <Step title="Активируйте модель">
    Нажмите **Сохранить**, затем **Активировать** рядом с новой записью.
  </Step>
</Steps>

<Tip>
Для лучших результатов с локальными моделями используйте модели с параметрами от 7 млрд. Модели с параметрами от 13 млрд и выше, как правило, дают более стабильные результаты оценки. Доступные варианты смотрите в [библиотеке Ollama](https://ollama.com/library).
</Tip>

## Файл инициализации models.yml

Doc Reviewer читает `models.yml` из папки рядом с `doc-reviewer.exe` один раз — при первом запуске — и использует его для заполнения начального списка моделей. После этого `models.yml` не влияет на работу приложения. Чтобы добавить или изменить модели после первого запуска, используйте **Настройки → Модели** в интерфейсе.

<Note>
Если вы разворачиваете Doc Reviewer на новом компьютере и хотите заранее настроить модели, поместите файл `models.yml` рядом с `.exe` до первого запуска приложения.
</Note>
