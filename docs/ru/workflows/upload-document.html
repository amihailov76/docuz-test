Загрузка документа и просмотр обнаруженных инструкций

При загрузке документа Doc Reviewer разбирает его структуру, делит на разделы и автоматически классифицирует каждый раздел как инструкцию, возможную инструкцию или обычный контент. Вы просматриваете и корректируете классификации до запуска оценки.

ℹ️Note

Поддерживаемые форматы файлов: PDF, DOCX, Markdown (.md) и TXT.

Загрузка документа​

1
Откройте страницу Проекты

Перейдите в Проекты в боковом меню и откройте проект, в который хотите добавить документ. Если проект ещё не создан, создайте его сначала — привязка документа к проекту позволяет LLM использовать контекст продукта при оценке.

2
Загрузите файл или перетащите его

На странице проекта нажмите Загрузить документ или перетащите файл в область загрузки. Вы также можете загрузить файл из заголовка страницы Оценка, если предпочитаете работать вне проекта.

3
Выберите файл

Выберите файл в формате PDF, DOCX, .md или .txt. Файлы в неподдерживаемых форматах сразу отклоняются с сообщением об ошибке.

4
Разрешите конфликты при необходимости

Если документ с таким же именем уже существует, приложение показывает диалог конфликта. Выберите один из вариантов:

  • Заменить — удалите существующий документ со всеми результатами оценки и загрузите новый файл.
  • Сохранить оба — загрузите новый файл как отдельную запись документа с тем же именем.
5
Дождитесь разбора и классификации

Doc Reviewer разбирает файл, извлекает иерархию разделов и запускает автоматическое обнаружение инструкций для каждого раздела. Для большинства документов этот шаг занимает несколько секунд. В процессе разбора отображается индикатор прогресса.

6
Просмотрите дерево документа

После завершения разбора документ открывается на странице Оценка. На левой панели отображается дерево документа со всеми обнаруженными разделами. Каждый раздел показывает значок классификации: инструкция, возможная инструкция или не инструкция.

Нажмите на любой раздел, чтобы просмотреть его содержимое на панели деталей. Вы можете изменить классификацию раздела вручную, нажав на значок и выбрав другое значение. Вы также можете переключить, включён ли раздел в следующую оценку.

💡Tip

Для наиболее точных результатов оценки привяжите документ к проекту с заданным контекстом продукта. Контекст продукта сообщает LLM название, аудиторию и ключевую терминологию продукта — это повышает точность оценки специализированных инструкций. Смотрите Проекты → Сгенерировать контекст на странице проекта.

Что происходит после загрузки​

После успешной загрузки Doc Reviewer:

  • Сохранил иерархию разобранных разделов в локальной базе данных SQLite
  • Присвоил каждому разделу одну из трёх классификаций:
    • инструкция — раздел явно является пошаговой процедурой
    • возможная инструкция — раздел может содержать инструкции (помечен для проверки)
    • не инструкция — раздел является вводным, концептуальным или справочным материалом
  • Сделал документ доступным для оценки

По умолчанию в оценку включаются только разделы с классификацией инструкция или возможная инструкция. Вы можете вручную включать или исключать отдельные разделы с помощью переключателя в дереве документа.
