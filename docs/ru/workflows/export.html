Экспорт результатов оценки в Excel

После завершения оценки вы можете экспортировать результаты в файл Excel (.xlsx). Экспорт создаёт структурированную таблицу, которую удобно передавать команде, отслеживать в трекере задач или прикладывать к отчёту по контролю качества.

Экспорт результатов​

1
Завершите оценку

Убедитесь, что хотя бы один раздел с инструкцией в документе прошёл оценку. Функция экспорта доступна только при наличии результатов оценки.

2
Нажмите «Экспорт» на странице документа

Откройте документ на странице Оценка. Нажмите кнопку Экспорт в заголовке документа. Файл скачивается немедленно — диалог не отображается.

3
Откройте скачанный файл

Файл сохраняется с именем файла документа в качестве базового имени, например installation-guide_review.xlsx. Откройте его в Excel, LibreOffice Calc или любом совместимом приложении для работы с таблицами.

Содержимое XLS-файла​

Экспортируемый файл содержит один лист Результаты оценки. Первая строка — заголовок с синим фоном. Каждая последующая строка представляет один оценённый раздел с инструкцией.

СтолбецСодержимое
РазделЗаголовок раздела с инструкцией, как он отображается в дереве документа
СтраницаНомер страницы в исходном документе (пусто для веб-документов)
ОценкаЦветовая оценка в виде метки: Хорошо, Замечания, Проблемы или Критично
РекомендацииВсе рекомендации LLM для этого раздела, каждая с префиксом в виде названия критерия и при необходимости примером

Столбец Оценка использует фоновые цвета ячеек, соответствующие цветовой шкале в приложении: зелёный, жёлтый, оранжевый или красный.

ℹ️Note

В экспорт включаются только разделы с результатами оценки. Разделы с классификацией не инструкция, разделы, отключённые от оценки, и разделы, пропущенные из-за ошибок, не включаются.

Сценарии использования​

  • Поделиться результатами с командой — передайте XLS редакторам, менеджерам продукта или специалистам по контролю качества, у которых не установлен Doc Reviewer.
  • Отслеживать задачи в трекере — скопируйте строки из таблицы в баг-трекер или доску проекта, чтобы создать задачи по правке документации.
  • Включить в отчёт по контролю качества — приложите файл к чек-листу релиза или отчёту аудита документации как подтверждение охвата оценки.
💡Tip

Если вы хотите сравнивать результаты между версиями документа в таблице, экспортируйте результаты после каждой оценки и храните файлы рядом. Для сравнения версий внутри приложения используйте снимки — они дают структурированный вид изменений без ручного управления файлами.
