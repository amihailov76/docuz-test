Проект — верхнеуровневый контейнер в Doc Reviewer. Вы создаёте один проект на продукт, добавляете все документы этого продукта и при желании генерируете контекст продукта — краткое описание, которое LLM читает перед оценкой каждой инструкции. Работа в рамках проектов позволяет организовать документы и сделать результаты оценки точнее и последовательнее.

## Что содержит проект

Каждый проект хранит:

- **Название**, идентифицирующее продукт (например, «Firewall Manager» или «Identity Console»)
- Необязательный **контекст продукта** — описание на ~400–700 слов, сгенерированное LLM или написанное вручную
- Все **документы**, привязанные к этому проекту

Вы можете создать столько проектов, сколько у вас продуктов. Каждый документ принадлежит ровно одному проекту, и вы можете переместить его в любой момент.

## Иерархия: проект → документы → инструкции → оценки

При запуске оценки Doc Reviewer оценивает каждую инструкцию в каждом документе. Инструкции из разных документов одного проекта получают одинаковый контекст продукта, поэтому LLM работает с последовательной фоновой информацией.

## Контекст продукта

Контекст продукта — текстовое описание на ~400–700 слов о документируемом продукте. Он включается в запрос LLM при оценке каждой инструкции в проекте, давая модели фоновую информацию для понимания специализированных терминов, ролей пользователей и названий компонентов.

Типичный контекст продукта охватывает:

- Название и категорию продукта (например, «сетевое устройство безопасности», «платформа управления идентификацией»)
- Целевую аудиторию и её роли
- 10–15 ключевых терминов и их значение в контексте продукта
- Основные компоненты или подсистемы, упоминаемые в документации

### Автоматическая генерация контекста

Doc Reviewer генерирует контекст продукта автоматически, читая не-инструкционные разделы из документов проекта: вводные тексты, разделы глоссария, обзорные страницы и другой справочный материал. Чтобы сгенерировать контекст:

<Steps>
  <Step title="Добавьте документы в проект">
    Загрузите хотя бы один документ, содержащий вводные или справочные разделы. Генератор читает до 60 не-инструкционных разделов из всех документов проекта.
  </Step>
  <Step title="Откройте страницу проекта">
    Перейдите в **Проекты**, нажмите на проект и найдите поле **Контекст продукта**.
  </Step>
  <Step title="Сгенерируйте контекст">
    Нажмите **Сгенерировать контекст**. Doc Reviewer отправит не-инструкционные разделы в LLM и автоматически сохранит результат.
  </Step>
</Steps>

### Редактирование контекста вручную

Вы можете написать или отредактировать контекст продукта в любой момент, щёлкнув прямо в поле **Контекст продукта** на странице проекта. Изменения сохраняются без обновления отметки времени генерации, поэтому вы можете дорабатывать сгенерированный текст, не теряя информацию о том, когда он был создан.

<Tip>
Генерируйте контекст продукта до первой оценки. Без него LLM оценивает инструкции, опираясь только на текст инструкции и общие знания о предметной области, что снижает точность — особенно для специализированной терминологии продукта.
</Tip>

## Создание проектов и управление ими

Чтобы создать проект, перейдите в **Проекты** и нажмите **Новый проект**. Задайте название, которое чётко идентифицирует продукт — оно отображается во всём интерфейсе, включая заголовок документа на экране оценки.

Чтобы переименовать проект или обновить его контекст, откройте страницу проекта и отредактируйте поля напрямую.

<Note>
Удаление проекта удаляет все связанные документы и результаты оценки без возможности восстановления. Перед удалением экспортируйте нужные результаты.
</Note>
