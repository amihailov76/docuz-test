# Ссылки, сноски и подсказки

---

## Активные ссылки: общие правила

Активная ссылка — ссылка, по которой можно перейти к целевому объекту.

Общие правила:

- **Предлог** перед текстом ссылки — часть ссылки. Включайте его в выделение.
- Используйте «**см.**» вместо «смотрите». После «см.» ставьте **неразрывный пробел**.
- Текст ссылки должен соответствовать ожиданиям пользователя — ссылка должна вести туда, куда сообщает текст.
- Не злоупотребляйте ссылками: много ссылок на единицу текста затрудняет чтение.

| Хорошо | Плохо |
|--------|-------|
| …вы можете прочесть [на сайте производителя] | …вы можете прочесть [на сайте производителя] (предлог «на» вне ссылки) |
| Коды языков см.&nbsp;[на сайте стандарта ISO 639-2] | Коды языков смотрите [на сайте стандарта ISO 639-2] |

## Гиперссылки

Гиперссылки — активные ссылки на веб-сайты, адреса email, номера телефонов.

- Предпочитайте **официальные ресурсы** (не Википедию).
- Для представления ссылки на веб-страницу или файл используйте слово **«сайт»** — не «ссылка», «адрес», «страница», «ресурс».
- Пишите адреса сайтов **без www.** и **без https://** (в самом URL протокол должен сохраняться).

### Формат

| Ситуация | Формат |
|----------|--------|
| Без адреса | см. [на сайте производителя] |
| С адресом | см. на сайте [example.org] |

### Примеры

| Хорошо | Плохо |
|--------|-------|
| Полный список кодов доступен [на сайте производителя]. | Перейдите по ссылке https://help.example.net/1234567/code-list.html |
| Этот файл вы можете скачать [на сайте компании Microsoft]. | Этот файл можно скачать [здесь]. |
| Подробное описание см. на сайте [docs.example.com]. | Подробное описание см. по адресу docs.example.com |

## Перекрёстные ссылки

Перекрёстная ссылка — ссылка на контент в том же документе или домене.

### Формат перекрёстных ссылок

| Цель ссылки | Формат |
|-------------|--------|
| Раздел с номером | …вам нужно (см. раздел 3.4) |
| Приложение с буквой | …можно задавать поля (см. приложение Г) |
| Приложение без номера | …можно задавать поля (см. приложение) |
| Раздел без номера в том же документе | Роль определяется [набором привилегий] |
| Рисунок пронумерованный | …данные заполняются в его карточке (см. рисунок 55) |
| Рисунок не пронумерованный | …данные заполняются [в его карточке] |
| Таблица пронумерованная | …набор параметров (см. таблицу 5) |

**Правило:** если ссылка и целевой объект — в одном разделе, номер раздела в ссылке не указывается.

---

## Блок «См. также»

Блок перекрёстных ссылок на связанные разделы. Добавляйте, когда:

- нужно сослаться на раздел, не привязываясь к конкретному месту в тексте;
- текст перегружен повторяющимися ссылками на один раздел.

**Формат:**

- Разделы с номерами: «Название раздела (см. раздел №)»
- Разделы без номеров: «Название раздела»

Блок может состоять из одной ссылки. Сортировка: от более важных к менее важным, или в порядке изложения, или по алфавиту.

Не дублируйте в «См. также» ссылку из блока «В этом разделе».

---

## Блок «В этом разделе»

Одноуровневое мини-содержание раздела. Добавляйте для навигации, если и раздел, и его дочерние разделы присутствуют в содержании.

Блок **не нужен**, если:
- документ носит справочный характер (API, Release Notes, траблшутинг);
- общее содержание уже выполняет навигационную функцию (например, в онлайн-справке с интерактивным меню).

---

## Блок «В этой статье»

Мини-содержание из ссылок на подзаголовки. Добавляется в начале статьи для навигации. Не делайте блок из одной ссылки.

## Сноски и подсказки

Сноски и подсказки помещают второстепенную информацию, чтобы не перегружать основной текст.

> ⚠️ Используйте с осторожностью: пользователи практически не читают сноски и подсказки.

**Не выносите в сноски:** определения, предупреждения, шаги инструкций, команды, описания компонентов продукта.

### Когда сноска оправдана

| Ситуация | Пример |
|----------|--------|
| Уточнение данных в таблице | Методология расчёта, единицы измерения |
| Ссылка на стандарт без URL | Промышленный протокол IEC 60870-5-104 |
| Второстепенная техническая деталь | «В отдельных режимах допускается SHA-384» |
| Очень редкий кейс (<1% случаев) | Специфика Oracle Solaris + ZFS дедупликация |
| Юридическое уведомление | Уведомление о запрещённых в РФ организациях |

## Текстовые ссылки

Текстовая ссылка — ссылка, которая только на словах сообщает адрес (нельзя кликнуть).

Используйте, когда:
- создать активную ссылку технически невозможно (печатный документ без веб-версии);
- целевой объект расположен рядом (в соседнем абзаце) — активная ссылка избыточна.

Глагол «смотрите» сокращайте до «**см.**». После «см.» — неразрывный пробел.

### Формат текстовых ссылок на документы

| Ситуация | Формат | Пример |
|----------|--------|--------|
| В названии есть «Руководство», «Справочник», «Справка» | Без кавычек, с большой буквы | …в Руководстве администратора |
| Без таких слов в названии | Добавить слово «документ» | …в документе Начало работы XSpider 12.3 |

### Формат текстовых ссылок на разделы

- Название раздела — в кавычках-ёлочках («»), без строчного стиля.
- Сопровождается названием документа.
- Называйте всегда «раздел» — не «глава», не «подраздел».
- Не указывайте номер раздела (может измениться).

| Хорошо | Плохо |
|--------|-------|
| Cм. раздел «Установка модуля» Руководства по установке | Cм. подраздел «Установка модуля» в главе «Установка продукта» |
| Подробные инструкции см. в Руководстве по установке MaxPatrol VM | Подробные инструкции смотрите в «Руководстве по установке MaxPatrol VM» |
| Затем выполните инструкцию из раздела «Активация лицензии» Руководства администратора | Затем выполните инструкцию из главы **Активация лицензии** Руководства администратора |
