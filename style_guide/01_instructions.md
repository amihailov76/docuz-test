# Как писать инструкции

## Что такое инструкция

Инструкция — это документ, описывающий, как выполнить задачу. Цель инструкции — помочь пользователю выполнить задачу, а не описать продукт.

---

## Анализ задачи перед написанием

Перед написанием инструкции определите:

- **Цель пользователя** — что пользователь хочет достичь.
- **Уровень сложности** — к какому типу относится задача (см. ниже).
- **Роль** — для кого предназначена инструкция (администратор, оператор и т.д.).
- **Контекст** — при каких условиях выполняется задача.
- **Предусловия** — что должно быть сделано до начала.
- **Результат** — что будет по завершении.

### Уровни сложности задач

| Тип | Описание |
|-----|----------|
| Операционная | Простая, одношаговая или малошаговая задача |
| Локальная | Несколько связанных шагов в рамках одного компонента |
| Составная | Охватывает несколько компонентов или сценариев |
| Рабочий процесс (workflow) | Серия задач, объединённых общей целью |

### Декомпозиция

Сложную задачу разбивайте на подзадачи. Каждой подзадаче — отдельная инструкция или раздел.

---

## Общие требования к инструкции

- Пишите инструкцию от задачи пользователя, не от функций программы.
- Включайте только то, что нужно для выполнения задачи.
- Не описывайте очевидное.
- Не смешивайте инструкцию с концептуальным описанием — выносите теорию в отдельный раздел.

---

## Заголовок инструкции

Заголовок инструкции должен:

- Быть **отглагольным существительным** (называть действие, а не объект): «Настройка уведомлений», не «Уведомления».
- Быть **уникальным** в рамках документа.
- Быть **ориентированным на задачу** пользователя.

**Хорошо:** «Создание резервной копии базы данных»  
**Плохо:** «База данных»

---

## Вводный текст

Вводный абзац инструкции (если нужен) должен:

- Объяснять, **зачем** выполнять задачу или что она даёт.
- Быть кратким — не более 2–3 предложений.
- Не пересказывать заголовок.

**Формула вводной фразы:** «Чтобы [цель], [выполните следующие шаги / настройте X].»

Пример: «Чтобы получать уведомления об инцидентах, настройте почтовый адрес получателя.»

---

## Правила написания шагов

### Основные правила

- Каждый шаг начинается с **глагола в повелительном наклонении**: «Нажмите», «Введите», «Выберите».
- Один шаг = одно действие.
- Если шаг начинается с контекстного условия, сначала укажите контекст: «В поле **Имя** введите...» — не «Введите имя в поле **Имя**...»
- Не используйте пассивный залог: «Нажмите кнопку» — не «Кнопка нажимается».
- Не используйте когнитивные глаголы в шагах: «убедитесь», «проверьте мысленно» и т.п. — вместо этого описывайте конкретное действие.

### Условные шаги

Формат: **«Если [условие], [выполните действие].»**

Пример: «Если сервер недоступен, проверьте подключение к сети.»

Условие всегда стоит перед действием.

### Примечания к шагу

Примечание к шагу помещается **после** шага, к которому относится. Используйте блок **Note** / **Tip** / **Warning**.

Не выносите в примечание обязательные действия — они должны быть шагами.

### Промежуточные результаты

Если после шага что-то происходит в интерфейсе, опишите результат в **будущем времени**:

«Откроется окно настроек.»

### Итоговый результат инструкции

В конце инструкции (если нужно) опишите, что достигнуто, — в **прошедшем времени**:

«Вы создали резервную копию базы данных.»

## Структуры инструкций

### Одношаговая

Используется, когда задача выполняется за одно действие.

```
## Заголовок

Чтобы [цель], [действие].

Пример: Чтобы обновить список событий, нажмите кнопку **Обновить**.
```

### Последовательная

Стандартная пронумерованная инструкция.

```
## Заголовок

Чтобы [цель]:

1. [Шаг 1]
2. [Шаг 2]
3. [Шаг 3]

[Итоговый результат]
```

### Условная

Содержит ветки «если / иначе».

```
1. [Общий шаг]
2. Если [условие A]:
   - [Шаг A1]
   - [Шаг A2]
   
   Если [условие B]:
   - [Шаг B1]
```

### С несколькими действиями на шаге

Используется редко — когда действия неразделимы.

```
1. Нажмите кнопку **Создать** и в открывшемся окне введите имя.
```

### Вложенная

Содержит подшаги.

```
1. Откройте раздел **Настройки**.
   1.1. Перейдите на вкладку **Сеть**.
   1.2. В поле **Адрес** введите IP-адрес сервера.
2. Нажмите **Сохранить**.
```

### Ветвящаяся

Разные сценарии выполнения одной задачи.

```
## Заголовок

Порядок действий зависит от [критерия].

### Вариант A: [название]

1. ...

### Вариант B: [название]

1. ...
```

---

## Специфика инструкций для командной строки

- Команды оформляйте в блоке кода с указанием языка (`bash`, `powershell` и т.д.).
- После команды описывайте ожидаемый вывод, если он неочевиден.
- Переменные в командах выделяйте угловыми скобками: `<имя_пользователя>`.
