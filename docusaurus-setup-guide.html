Docusaurus: от локального проекта до GitHub Pages

Миграция документации с Mintlify на Docusaurus и деплой на GitHub Pages

Обзор проекта

Исходные данные: документация проекта Doc Reviewer в репозитории Mintlify на GitHub. Цель — развернуть её на Docusaurus локально, затем опубликовать на GitHub Pages.

1. Создание Docusaurus-проекта

Команды

# Создать проект (выбрать JavaScript при подсказке)
npx create-docusaurus@latest my-docs classic
cd my-docs

# Запустить локально
npm start

После старта сайт доступен на http://localhost:3000. Docusaurus поддерживает hot reload.

⚠️
Требования Node.js v18 или новее. Проверьте: node -v. Версии ниже 18 не поддерживаются Docusaurus v3.

2. Конфигурация сайта

Файл docusaurus.config.js — главный конфигурационный файл. Ниже ключевые настройки с пояснениями.

Для локальной разработки

const config = {
  title: 'Doc Reviewer',
  url:     'http://localhost:3000',
  baseUrl: '/',
  onBrokenLinks: 'warn',      // не прерывать сборку — удобно при миграции
  onBrokenMarkdownLinks: 'warn',
  // ...
};

Для GitHub Pages

const config = {
  url:              'https://amihailov76.github.io',
  baseUrl:          '/docuz-test/',   // имя репозитория
  organizationName: 'amihailov76',
  projectName:      'docuz-test',
  trailingSlash:    false,
  // ...
};
⚠️
Сложное место: baseUrl и внутренние ссылки Когда baseUrl не равен /, все внутренние ссылки и редиректы нужно строить через useBaseUrl(), иначе пути будут битыми на деплое.

Навбар: два языковых раздела вместо табов Mintlify

navbar: {
  items: [
    { type: 'docSidebar', sidebarId: 'enSidebar', label: 'English'  },
    { type: 'docSidebar', sidebarId: 'ruSidebar', label: 'Русский'  },
    { href: 'https://github.com/amihailov76/doc-reviewer', label: 'GitHub', position: 'right' },
  ]
}

3. Перенос документов

Mintlify и Docusaurus оба используют MDX, поэтому сами .mdx-файлы переносятся без изменений. Нужно только правильно разместить папки.

  1. Клонировать исходный репозиторий
    git clone https://github.com/amihailov76/mintlify-docs
  2. Скопировать папки с контентом
    Перенести en/ и ru/ из Mintlify-репо в папку docs/ Docusaurus-проекта.
  3. Удалить дефолтный контент
    Удалить из docs/ папки tutorial-basics/, tutorial-extras/ и файл intro.md — они созданы Docusaurus при инициализации и больше не нужны.
ℹ️
Mintlify-компоненты не сломают сборку сразу Если в MDX-файлах есть компоненты вроде <Card> или <Steps>, сборка не упадёт — Docusaurus просто не знает, что с ними делать. Но при рендере страниц появятся ошибки вида Expected component X to be defined. Решение — шаг 6.

4. Настройка сайдбара

Mintlify хранит навигацию в docs.json (секция navigation.tabs). В Docusaurus это sidebars.js с именованными сайдбарами — по одному на каждый язык.

// sidebars.js
const sidebars = {
  enSidebar: [
    { type: 'category', label: 'Get Started', collapsed: false,
      items: ['en/introduction', 'en/installation', 'en/quickstart'] },
    { type: 'category', label: 'Core Concepts',
      items: ['en/concepts/projects', 'en/concepts/documents',
              'en/concepts/evaluation', 'en/concepts/criteria'] },
    // ... Workflows, Configuration, Troubleshooting
  ],
  ruSidebar: [
    // зеркальная структура для ru/
  ],
};
module.exports = sidebars;
💡
Альтернатива ручному сайдбару Для быстрого старта можно использовать автогенерацию: { type: 'autogenerated', dirName: 'en' }. Docusaurus построит сайдбар из структуры папок. Но порядок страниц придётся регулировать числовыми префиксами в именах файлов (01-introduction.mdx).

5. Стили и фирменные цвета

Цвета из docs.json Mintlify переносятся в CSS-переменные Docusaurus. Файл: src/css/custom.css.

/* Светлая тема */
:root {
  --ifm-color-primary:          #2D7DD2;
  --ifm-color-primary-dark:     #2570C4;
  --ifm-color-primary-darker:   #2169BA;
  --ifm-color-primary-darkest:  #1A549A;
  --ifm-color-primary-light:    #3F8ADA;
  --ifm-color-primary-lighter:  #4892DE;
  --ifm-color-primary-lightest: #68A8E8;
}

/* Тёмная тема */
[data-theme='dark'] {
  --ifm-color-primary: #5EA8F0;
  /* ... более светлые оттенки для читаемости */
}
⚠️
Docusaurus требует семь оттенков Нельзя указать только один цвет — нужны все семь вариантов (primary, dark, darker, darkest, light, lighter, lightest). Используйте генератор палитры Infima.

6. Замена Mintlify-компонентов

Docusaurus позволяет зарегистрировать любые JSX-компоненты глобально через файл src/theme/MDXComponents.js. Все компоненты, найденные в репозитории:

<Card> / <CardGroup>
Карточки в CSS-сетке. Поддерживают prop href для кликабельных карточек и icon для эмодзи.
<Steps> / <Step>
Нумерованные шаги с вертикальной линией. Нумерация автоматическая через React.Children.
<Note> / <Tip> / <Warning> / <Info>
Информационные блоки. Используют классы Docusaurus alert — совместимы со светлой и тёмной темами.
<CodeGroup>
Кодовые блоки с вкладками. Метка вкладки берётся из metastring: ```text OpenAI.
<Tabs> / <Tab>
Контентные вкладки для произвольного MDX. Метка — из prop title.
<Accordion> / <AccordionGroup>
HTML <details>/<summary> со стилизованной стрелкой и слитыми рамками.
// src/theme/MDXComponents.js
import MDXComponents from '@theme-original/MDXComponents';
import DocTabs from '@theme/Tabs';       // переименовать, чтобы не конфликтовать!
import DocTabItem from '@theme/TabItem';

// ... определения компонентов ...

export default {
  ...MDXComponents,   // сохранить стандартные компоненты Docusaurus
  Card, CardGroup, Steps, Step,
  Note, Tip, Warning, Info,
  CodeGroup, Tabs, Tab,
  Accordion, AccordionGroup,
};
🔴
Конфликт имён: Tabs Mintlify использует компонент <Tabs>, и Docusaurus тоже экспортирует Tabs из @theme/Tabs. Если импортировать оба под одним именем — конфликт. Решение: импортировать Docusaurus-вкладки как DocTabs и использовать это имя внутри компонентов.
⚠️
alert--info отсутствует в Docusaurus Стандартные классы Docusaurus: alert--secondary, alert--success, alert--warning. Класса alert--info нет — его нужно добавить вручную в custom.css.

7. Домашняя страница → редирект

По умолчанию Docusaurus показывает лендинг (src/pages/index.js). Чтобы сразу открывалась нужная страница документации — заменить содержимое на редирект.

// src/pages/index.js
import useBaseUrl from '@docusaurus/useBaseUrl';
import { Redirect } from '@docusaurus/router';

export default function Home() {
  return <Redirect to={useBaseUrl('/docs/ru/introduction')} />;
}
🔴
Самая частая ошибка: пропущен useBaseUrl <Redirect to="/docs/ru/introduction" /> работает локально (baseUrl = /), но ломается на GitHub Pages (baseUrl = /docuz-test/). Путь без useBaseUrl игнорирует baseUrl и ведёт на несуществующий адрес. Всегда оборачивайте путь в useBaseUrl().

8. Деплой на GitHub Pages

  1. Инициализировать git и связать репозиторий 
    git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/amihailov76/docuz-test.git
git push -u origin main
  2. Создать GitHub Actions workflow
    Файл должен находиться строго по пути .github/workflows/deploy.yml. 
    name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
  workflow_dispatch:
jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: npm }
      - run: npm ci
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./build
  3. Включить права на запись для GitHub Actions
    Settings → Actions → General → Workflow permissions → Read and write permissions → Save.
  4. Настроить GitHub Pages
    Settings → Pages → Source: Deploy from a branch → Branch: gh-pages, folder: / (root) → Save.
  5. Запушить и дождаться деплоя
    После пуша в main Actions создаёт ветку gh-pages и деплоит сайт. Прогресс — во вкладке Actions репозитория. Сайт появляется через 1–2 минуты.
🔴
Частая ошибка: deploy.yml в .git\ вместо .github\ Папка .git\ — служебная папка Git, она никогда не попадает в репозиторий. Правильный путь: .github\workflows\deploy.yml (папка .github рядом с src\, docs\, package.json).

Типичные ошибки и решения

Ошибка Причина Решение
Expected component X to be defined Mintlify-компонент не зарегистрирован в Docusaurus Добавить компонент в src/theme/MDXComponents.js
Редирект ведёт на 404 baseUrl не учтён в пути редиректа Использовать useBaseUrl('/docs/...')
Ветка gh-pages не создаётся Нет прав на запись у GITHUB_TOKEN Settings → Actions → Workflow permissions → Read and write
GitHub Pages показывает 404 Страница GitHub Pages указывает не на ту ветку Settings → Pages → Branch: gh-pages
Конфликт имён Tabs Mintlify Tabs и Docusaurus Tabs — разные компоненты Импортировать Docusaurus-вариант как DocTabs
Сломанные внутренние ссылки на GitHub Pages baseUrl не подставляется в ссылки Использовать относительные ссылки или useBaseUrl()
deploy.yml не попадает в репозиторий Файл помещён в .git\ вместо .github\ Переместить в .github\workflows\deploy.yml
